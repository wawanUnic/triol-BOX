1. Первоначальная проблема: невозможно забрать приватный ключ с роутера
Ты пытался скачать приватный ключ Dropbear через FileZilla, но:

FileZilla требует SFTP,

Dropbear на OpenWrt не поддерживает SFTP,

поэтому FileZilla выдавал ошибку
ash: /usr/libexec/sftp-server: not found.

Мы выяснили, что:

Dropbear поддерживает SSH и SCP,

но на старых сборках SCP часто вырезан.

2. Попытка скачать ключ через pscp.exe
Ты пробовал:

Kod
pscp.exe -P 23 root@192.168.112.48:/etc/dropbear/id_rsa C:\openwrt
Но после ввода пароля появлялось:

Kod
ash: /usr/libexec/sftp-server: not found
FATAL ERROR: Received unexpected end-of-file from server
Это означало, что:

SCP на роутере отсутствует,

PSCP пытается переключиться в SFTP,

SFTP отсутствует,

соединение рвётся.

3. Попытка вывести ключ через cat
Ты вывел файл:

Kod
cat /etc/dropbear/id_rsa
Но содержимое оказалось бинарным мусором, а не ключом:

нет заголовка -----BEGIN DROPBEAR RSA PRIVATE KEY-----,

нет base64,

файл не читается.

Мы установили, что:

Dropbear в твоей прошивке не умеет генерировать пользовательские ключи,

dropbearkey создаёт повреждённый бинарный файл.

Это типично для TL‑WR740N с 4 MB флеша.

4. Правильное решение: генерировать ключ на компьютере
Мы перешли на рабочую схему:

На ПК создать ключ:

Kod
ssh-keygen -t rsa -b 2048 -f C:\keys\openwrt_key
Получить публичный ключ:

Kod
cat C:\keys\openwrt_key.pub
Загрузить публичный ключ на роутер.

5. Загрузка публичного ключа через веб‑морду
Ты заметил, что LuCI позволяет загружать файлы — это действительно удобный способ.

В LuCI:

System → Administration → SSH Access → Authorized Keys → Upload SSH Key File

Туда загружается публичный ключ (openwrt_key.pub).

Dropbear автоматически создаёт:

Kod
/etc/dropbear/authorized_keys
6. Конвертация приватного ключа в формат PuTTY
PuTTY не понимает OpenSSH‑ключи, поэтому:

Открыли PuTTYgen.

Conversions → Import key.

Выбрали C:\keys\openwrt_key.

Сохранили как openwrt_key.ppk.

Теперь PuTTY может использовать ключ.

7. Подключение по ключу
В PuTTY:

Host: 192.168.112.48

Port: 23

Connection → SSH → Auth → Credentials → Private key file: openwrt_key.ppk

После этого SSH‑вход работает без пароля.

8. Отключение входа по паролю в SSH через веб‑морду
В LuCI:

System → Administration → SSH Access → Dropbear Instance

Мы отключили:

Allow SSH password authentication

Allow root logins with password

Теперь SSH принимает только ключи.

9. Важное уточнение: веб‑морда всегда использует пароль
LuCI не поддерживает SSH‑ключи, поэтому:

пароль root остаётся для веб‑интерфейса,

но SSH теперь работает только по ключу.

Это нормальная и безопасная схема.

10. SSH‑туннель как способ безопасно работать с веб‑мордами без TLS
Так как роутер не поддерживает HTTPS, мы настроили SSH‑туннель:

Kod
ssh -p 23 -L 8080:192.168.112.48:80 root@192.168.112.48
Теперь:

браузер открывает http://localhost:8080,

весь трафик идёт внутри SSH‑шифрования,

веб‑морда фактически защищена.

Полная инструкция по настройке SSH‑доступа и ключевой аутентификации на старом роутере OpenWrt (TL‑WR740N)
1. Исходная ситуация и проблемы
На роутере OpenWrt 19.07 (TL‑WR740N):

нет поддержки SFTP;

SCP отсутствует или вырезан;

Dropbear генерирует повреждённый приватный ключ (бинарный мусор);

веб‑морда работает только по HTTP (нет TLS);

SSH работает на нестандартном порту (23).

Из‑за этого невозможно скачать ключ через FileZilla или PSCP, а встроенный Dropbear не создаёт корректный ключ.

2. Генерация SSH‑ключа на компьютере
Так как роутер не может создать корректный ключ, ключ создаётся на ПК.

Команда для Windows (PowerShell или Git Bash):
```
ssh-keygen -t rsa -b 2048 -f C:\keys\openwrt_key
```
После этого появятся два файла:

openwrt_key — приватный ключ (хранится только у вас);
openwrt_key.pub — публичный ключ (загружается на роутер).

3. Загрузка публичного ключа на роутер через веб‑морду LuCI
Это самый надёжный способ, так как SCP/SFTP не работают.

Путь в LuCI:
System → Administration → SSH Access → Authorized Keys

Действия:
Нажать Upload SSH Key File.

Выбрать файл openwrt_key.pub.

Сохранить изменения.

После этого на роутере появится файл:
```
/etc/dropbear/authorized_keys
```
Dropbear теперь знает ваш публичный ключ.

4. Конвертация приватного ключа в формат PuTTY
PuTTY не понимает OpenSSH‑ключи, поэтому нужно конвертировать.

Действия в PuTTYgen:
Открыть PuTTYgen.

Меню Conversions → Import key.

Выбрать C:\keys\openwrt_key.

Нажать Save private key.

Сохранить как openwrt_key.ppk.

Теперь PuTTY может использовать ключ.

5. Подключение к роутеру по ключу
В PuTTY:

Host Name: root@192.168.112.48

Port: 23

Connection → SSH → Auth → Credentials → Private key file:
C:\keys\openwrt_key.ppk

После подключения вы должны попасть в shell без пароля.

6. Отключение входа по паролю в SSH (Dropbear)
Чтобы SSH принимал только ключи, а пароль был отключён:

Через веб‑морду LuCI:
System → Administration → SSH Access → Dropbear Instance

Снять галочки:

Allow SSH password authentication

Allow root logins with password

Сохранить.

Теперь SSH принимает только ключи.

7. Важное замечание: веб‑морда LuCI всегда использует пароль
LuCI не поддерживает SSH‑ключи, поэтому:

пароль root остаётся для входа в веб‑интерфейс;

SSH работает только по ключу.

Это нормальная и безопасная схема.

8. Настройка SSH‑туннеля для безопасного доступа к веб‑морде
Так как роутер не поддерживает HTTPS, веб‑морда остаётся HTTP.
Чтобы защитить её, используется SSH‑туннель.

Команда:
Kod
ssh -p 23 -L 8080:192.168.112.48:80 root@192.168.112.48
Теперь:

браузер открывает http://localhost:8080;

весь трафик шифруется SSH;

веб‑морда фактически защищена.

9. Проблема с отсутствием оболочки в PuTTY
Если PuTTY подключается, но shell не появляется:

Причина — включена опция:

Connection → SSH → Don’t start a shell or command at all

Решение:
Снять галочку → сохранить → подключиться снова.

10. Итоговая схема работы
После всех настроек:

SSH работает только по ключу.

Парольный вход в SSH отключён.

Веб‑морда доступна только через SSH‑туннель.

Приватный ключ хранится только на ПК.

Публичный ключ хранится в /etc/dropbear/authorized_keys.

Dropbear не генерирует ключи — это нормально для старых роутеров.
